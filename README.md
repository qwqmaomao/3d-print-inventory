# 3D 打印工作台 v2.1

一个可直接打开 `index.html` 使用的中文静态网页，用于记录 3D 打印耗材库存、图片、消耗记录、代打报价、打印队列和常用工作台入口。项目无后端，数据保存在当前浏览器本地。

## 使用方式

直接用浏览器打开 `index.html`。

本地数据位置：

- `filament_inventory_v1`：耗材库存
- `filament_consumption_v1`：使用记录 / 库存流水账
- `filament_queue_v1`：打印队列
- `filament_devices_v1`：轻量设备管理
- `filament_cost_settings_v1`：报价/成本参数
- `filament_trash_v1`：回收站
- `filament_duplicate_ignores_v1`：重复检测忽略记录
- `filament_duplicate_negative_rules_v1`：重复检测负向规则
- `filament_restock_settings_v1`：补货设置
- `filament_label_settings_v1`：标签尺寸和二维码内容设置
- IndexedDB `filament_inventory_assets_v1`：WPS/手动上传图片

## v2.1 功能

- 默认进入“工作台”主页，集中显示库存总览、低库存、待打印任务、疑似受潮、需要盘点和最近动态
- 新增快捷入口：新增耗材、添加记录、扫码查找、打印标签、进入排班、导入 Excel、完整 ZIP 备份
- 新增“实用工具中心”，集中放置报价计算、补货清单、二维码标签、库存盘点、数据体检、扫码查找、Excel 导入、完整备份和回收站入口
- 明确区分“新增耗材”和“添加记录”：前者用于入库，后者用于消耗或库存变化记录
- 使用记录升级为库存流水账，显示类型、变化量、变化前重量、变化后重量、关联任务、成本和备注
- 流水类型覆盖入库、打印消耗、扫码消耗、队列扣减、失败损耗、盘点修正、手动调整、位置变更、开封和干燥记录
- 新增“统计分析中心”，按库存结构、最近 7/30/90 天消耗、项目成本、失败损耗和补货压力做轻量分析
- 打印队列升级为轻量排班：支持优先级、预计开始/完成时间、关联设备、库存检查、失败后重新排队和完成后成本摘要
- 新增“设备管理”：记录设备名称、型号、喷嘴直径、支持材料、状态、位置和备注；不接入真实打印机
- JSON/ZIP 备份恢复包含设备列表，旧备份没有设备数据时自动按空列表兼容
- 新增“数据安全中心”：显示当前数据版本、图片引用、流水、队列和设备概览
- JSON/ZIP 恢复前会先显示覆盖预览，确认后才写入 LocalStorage；ZIP 会提示备份内缺失的图片文件数量
- 新增图片缺失检查，检查库存和回收站 imageKey 是否能在 IndexedDB 找到原图
- 新增单卷迁移只读预览，将当前聚合库存预览为 `FIL-0001-01` 这类实体卷编号，不改写主数据
- 保留 v1.7-v2.0 的库存、图片、导入导出、备份、打印队列、补货、体检、盘点、标签、二维码、扫码快捷操作和重复检测 2.0

- WPS/Excel `.xlsx` 导入，兼容项目内 `耗材储存.xlsx`
- WPS `DISPIMG` 图片提取并保存到 IndexedDB
- 新增、编辑、删除耗材；删除进入回收站，可恢复
- 库存表格支持品牌、材料、颜色、卷数、剩余、位置、状态快速编辑
- 每卷默认 1000g，库存进度条按 `剩余重量 / (卷数 × 每卷重量)` 显示
- 手动添加图片、预览图片、清除图片
- 消耗记录保存后扣减库存
- 打印队列支持一个任务添加多种耗材
- 队列支持预计消耗和实际消耗，完成时按实际消耗扣库存
- 队列状态：待打印、准备中、打印中、暂停、已完成、失败、取消
- 失败任务可记录原因，并可选择生成消耗记录或重新加入队列
- 代打报价包含材料、电费、机器磨损、人工、模型处理、后处理、包装、失败损耗、利润倍率和最低起步价
- 补货清单：按材料阈值生成建议采购列表，并支持复制文本和导出 `.xlsx`
- 筛选结果导出 `.xlsx`
- 疑似重复检测 2.0：弱化品牌/基础材料的基础分，线径仅作参考不参与评分，使用强证据、hard cap、字段对比、判断原因和合并预览区分“同类耗材”和“疑似重复”
- 重复检测支持 90+、75+、60+ 分数筛选，以及隐藏/显示已忽略项
- 重复检测加入材料/颜色结构化解析和 hard cap，避免 ABS 与 ABS-GF、湖蓝与深蓝被误判为高重复；库存体检同步尊重忽略和负向规则
- 支持“不是重复，记住这个判断”，后续自动避开类似材料或颜色误判
- 合并前可预览合并后的品牌、材料、颜色、线径、卷数、剩余重量、购买价格、备注、图片策略和记录迁移
- 新增库存体检：检查重复、缺少品牌/材料/颜色、超容量、状态与剩余矛盾、缺图、开封久未干燥等问题
- 库存盘点模式：填写实际称重，保存后生成盘点修正记录
- 干燥/受潮管理字段和建议干燥高亮
- 自动生成稳定耗材编号 `FIL-0001`
- 明文 + 二维码标签打印，支持小/中/大标签尺寸
- 扫码查找或手动输入 `FIL-xxxx` 定位耗材
- 扫码快捷操作：查看详情、快速消耗、自定义扣减、标记开封、更新位置、进入盘点、加入队列

## 后续方向

单卷管理暂不执行破坏性迁移。当前 v2.1 只提供只读迁移预览：在聚合库存下预览 `FIL-0001-01`、`FIL-0001-02` 这类实体卷编号，后续确认迁移方案后再分别记录每卷剩余重量、开封状态、干燥状态和具体位置。扫码解析函数已预留 `FIL-0001-01` 这类单卷编号格式。

## 开发宪章

后续开发遵循 `.specify/memory/constitution.md`：

- 优先保持现有库存、导入、图片、备份、回收站、补货、队列、报价、重复检测、标签和扫码功能稳定。
- 新功能先写清楚用户场景、边界条件和验收标准，再改代码。
- 不随意大重构，不引入不必要依赖；确需依赖时必须本地打包并说明用途。
- 涉及数据结构、LocalStorage、IndexedDB、导入或备份格式时必须说明兼容和迁移策略。

## 备份说明

JSON 备份：

- 适合快速备份文字数据
- 包含库存、消耗记录、打印队列、成本设置、回收站等
- 只保存图片 `imageKey`，不内嵌原图

完整 ZIP 备份：

- ZIP 内包含 `data.json` 和 `images/`
- 会从 IndexedDB 读取耗材图片并一起保存
- 适合迁移到另一台电脑或另一个浏览器

## 表格导入

优先兼容 `耗材储存.xlsx` 的字段：

- `厂家` -> 品牌
- `材质` -> 材料
- `颜色` -> 颜色
- `重量/卷数` -> 当前剩余重量和卷数
- `备注图` -> WPS 图片
- `备注` -> 备注
- `位置序列号` -> 位置

其他 Excel 表格继续使用常见中文列名自动映射。

## 开源模块

- `vendor/jszip.min.js`：JSZip，用于读取 WPS 图片和完整 ZIP 备份
